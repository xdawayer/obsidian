# 《逻辑创新思维法》精简版

## 一句话总结

创新不是天才的灵光一闪，而是可以被训练的技能——用逻辑框架驾驭创新方向，用创新突破逻辑局限，两者交替运行，产出既有新意又经得起验证的方案。

---

## 核心公式

逻辑基础 + 发散生成 + 逻辑筛选 + 迭代优化 = 可靠的创新

---

## 逻辑创新五步循环

| 步骤 | 主导思维 | 核心动作 |
|------|---------|---------|
| 1. 界定问题 | 逻辑 | 分析、拆解、定义 |
| 2. 生成方案 | 创新 | 发散、联想、组合 |
| 3. 筛选方案 | 逻辑 | 评估、过滤、排序 |
| 4. 优化方案 | 创新 | 改进、变体、整合 |
| 5. 验证方案 | 逻辑 | 检验、测试、确认 |

关键原则：解决正确的问题比正确地解决问题更重要。

---

## 四种创新模式

- 组合创新：A + B = C（智能手机 = 电话 + 相机 + 电脑）
- 迁移创新：领域X的方案迁移至领域Y（自然界钩刺结构 - 魔术贴）
- 逆向创新：把某属性反转（荷兰式拍卖：从高价往下降）
- 减法创新：去掉某部分（无印良品：去掉多余品牌装饰）

---

## 发散与收敛

发散阶段四条铁律：
1. 延迟判断：不评价好坏
2. 追求数量：数量是质量的前提
3. 鼓励搭便车：在别人想法上叠加
4. 欢迎疯狂想法

收敛阶段三原则：
1. 建立评估标准
2. 分层过滤
3. 保护弱苗

关键：发散和收敛必须在时间上分离。同时进行，就像同时踩油门和刹车。

---

## 五种思维定式

1. 功能固着：只看到常规用途
2. 经验陷阱："我们一直这么做"
3. 权威定式：专家说的就是对的
4. 从众定式：大家都这么想
5. 非此即彼：看不到第三种可能

打破方法：
- 强制关联法：随机选取两个无关事物，强迫找联系
- 假设挑战法：列出假设，逐一追问"如果不成立呢"
- 视角转换法：如果你是竞争对手/用户的孩子/在火星上

## 思维工具速查

| 场景 | 推荐工具 |
|------|---------|
| 需要大量新想法 | 头脑风暴 |
| 团队讨论陷入对立 | 六顶思考帽 |
| 系统改进现有产品 | 奥斯本检核表（九个问题） |
| 面对技术性矛盾 | TRIZ |
| 理清复杂关系 | 思维导图 |

奥斯本检核表九问：
1. 能否替代？
2. 能否组合？
3. 能否调整？
4. 能否放大？
5. 能否缩小？
6. 能否挪用？
7. 能否重排？
8. 能否反转？
9. 能否消除？

---

## 创新者的心智模型

1. 成长型心智：相信创新能力可以训练
2. 容错型心智：接受失败是必要成本
3. 好奇型心智：对"理所当然"保持追问
4. 整合型心智：追问"能不能两个都要"

方法论解决"怎么做"，心智模型解决"敢不敢做"。后者往往更关键。

## 行动清单

本周：
- 面对任何问题，先列出所有隐含假设
- 找到一个可以被挑战的假设

本月：
- 用奥斯本检核表完整分析一个工作问题
- 记录并复盘使用效果

持续练习：
- 每天10分钟，从不熟悉的领域找一个案例
- 思考能否迁移到你的工作中
- 积累"跨领域素材库"

## 费曼检验

向朋友解释这本书：

"大多数人觉得创新是天才的灵光一闪。丁敏说这搞反了——创新是可以训练的技能，关键工具恰恰是逻辑。

想象一堆乐高积木，每块都是旧的。创新就是用旧积木拼出新造型。逻辑帮你三件事：弄清想拼什么（定义问题）、检查拼法是否稳固（验证方案）、选出最好的设计（筛选方案）。

操作分五步：用逻辑定义问题，用创新大量生成方案，用逻辑筛选最优，用创新再优化一轮，用逻辑最后验证。像呼吸一样，吸气（发散）和呼气（收敛）交替进行。

但最重要的：在学方法之前，先检查你的心态——你是否相信创新是可以学会的？如果不信，再好的方法也是摆设。"
